01The problem

A caller posing as CBI, customs or police manufactures a fake case, isolates the victim ("don't tell anyone, don't hang up"), and over 30–60 minutes coerces a transfer of life savings. It is a government-flagged, national-scale problem in India.

Why existing defences failWhat actually happens
The number is disposableFresh SIMs, spoofed IDs, WhatsApp calls. By the time a number is blacklisted it has been discarded — number-reputation is always one step behind.
The victim authorises the paymentTo a bank's fraud engine the transfer looks perfectly legitimate. The fraud signal lives in a conversation the bank cannot see.
The victims are the least-served usersElderly, non-English-speaking, low-literacy and disabled people — exactly who English-first, app-fluent tools fail.

02Where it intervenes

Scams follow a stable coercion grammar. DhvaniShield tracks that arc and fires before money moves — not after.

03Architecture

The safety invariant. There is no SAFE class. The output space is HIGH_RISK / UNCERTAIN / NO_PATTERN, and learned layers may only escalate — so no model error can ever produce a false "you're safe". A false alarm costs one verification call; a false reassurance costs a life's savings. The asymmetry is built into the architecture.

04The Manipulation Meter

Instead of a black-box score, the user sees what is being done to them. Real output for: "This is CBI, a case is filed, do not tell anyone, transfer money now."

07Built for the people scammers actually target

Most scam tools assume a sighted, literate, English-reading, app-fluent user — the exact person scammers avoid. Six accessibility profiles, each enforced as a test:

ProfileHow the single alert is delivered
Blind / low-visionspoken script (short sentences, action repeated) plus a distinct audio earcon — no visual dependency
Deaf / hard-of-hearingpictogram card plus a distinct haptic vibration pattern — no audio dependency
Colour-blindmeaning carried by shape + word ("STOP"), never colour alone
Low-literacyscam icon plus a grade-1 easy-read line
Cognitive / elderlyone step only, the family code word, and one tap to a trusted person
The family code word. A household agrees one secret word; any caller claiming to be family must produce it. It costs nothing, needs no technology, and defeats even AI voice cloning — which no detector can reliably do.

08Privacy, security and compliance

Privacy by architecture

  • No database, no data at rest. Content processed and deleted within the request.
  • Telemetry and receipts store labels and hashes only — never text.
  • On-device inference: no cloud AI call; nothing leaves the phone.
  • DPDP Act 2023 obligations satisfied largely by construction.

Enterprise controls (tested)

  • Multi-key per-tenant auth with zero-downtime rotation.
  • Twilio webhook signature validation; Redis-backed rate limiting.
  • Size caps, security headers, dependency scanning in CI.
  • Startup audit refuses to run silently insecure.

Duty-of-care receipt. Every assessment issues a signed, content-free receipt proving a specific message was assessed and the user warned — storing only a keyed hash. For an institution under pressure to reimburse scam losses, this turns a detector into a compliance asset with zero new privacy liability.

10Limitations and roadmap

Stated, not hidden

  • Validated on real messages; live call validation needs a consented pilot.
  • Manipulation engine is a research direction at AUC 0.66 on real data.
  • Roughly a quarter of genuinely ambiguous authority calls get "verify independently" — deliberate abstention, since a real and a fake police call are textually identical.
  • Calibration under distribution shift and compositional reasoning remain open problems.

Roadmap

  1. Pull-based pilot — a WhatsApp number people forward to. Zero install, zero call access. Deployable today.
  2. Close the data flywheel — consented labels build a proprietary real-scam corpus; accuracy compounds with use.
  3. On-device call screening — safe only because the collect-nothing foundation exists first.
  4. Institutional deployment — bank / telco / helpline, gated on third-party security and DPDP audit.

11Run it yourself

cd dhvanishield_v6_deploy_1
pip install -r requirements.txt
set PYTHONUTF8=1

python tests/run_all.py          # 22 suites - all must pass

python check.py "This is CBI, you are under digital arrest, transfer money now"
python check.py --ambient "Hi, your delivery is scheduled for Friday"        # stays silent
python check.py --ambient --profile=blind "This is CBI, transfer money now"  # accessibility

uvicorn server:app --port 8000   # /v1/check  /v1/screen  /v1/analyze  /v1/manipulation
Why this matters. It is not a better spam filter. It is a second opinion that reaches the person who needs it, at the moment they need it, in the language they speak, through the sense they can perceive — and it is built so it can never be the thing that tells a victim a scam was safe.
